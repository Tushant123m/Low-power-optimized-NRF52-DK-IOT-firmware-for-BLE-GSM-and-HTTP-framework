Many of us who are new to nordic boards often struggle to understand the libraries and their funcitons, event based variables and their use, as there are lots of macros which are new to standard arduino/esp32/stm32 developers.
This is my internship project where, i used NRF52 DK, GSM, API server and ble to operate the board in two modes, config and normal modes, in where the config mode is when we have to update the flash memory of the board with some new data. That new data will have to be regularly sent to server every 15 minutes.
So, in config mode, the BLE is used to send new config data from phone nordic app to DK directly, and stored in flash, and then this data is read from flash sent from gsm to http servers continuously in a loop. This continuous sending is done in normal mode, and hence name normal.
